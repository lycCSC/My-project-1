# Dialogue System Test Plan

## Core Flow

- 进入 `Scene01` 后，示例 NPC 出现在玩家附近。
- 玩家接近 NPC 后出现交互提示。
- 按交互键或交互按钮后打开对话面板。
- 第一段文本正确显示说话人和正文。
- 点击继续后进入下一节点。
- 出现分支节点时展示多个选项按钮。
- 点击不同选项后跳到对应节点。
- 结束节点后关闭对话框并恢复游戏。

## Pause Behavior

- 对话开始后，敌人停止追击、攻击和巡逻动画推进。
- 对话开始后，不再生成新敌人。
- 如果场景中已有子弹，子弹在对话期间不再推进。
- 对话期间玩家不能移动、攻击、跳跃和冲刺。
- 对话期间玩家对话动画继续播放。

## Resume Behavior

- 对话结束后 `Time.timeScale` 恢复正常。
- 玩家恢复到非 `talk` 状态。
- 敌人和刷怪继续正常运行。
- 关闭对话后交互提示按距离重新出现或隐藏。

## Regression Checks

- 菜单暂停仍可正常打开和关闭。
- 当菜单已暂停时，不会因为结束对话而误恢复整场游戏。
- 玩家受击、冲刺冷却、敌人受击等现有逻辑在无对话时行为不变。
- 未进入 NPC 范围时，交互输入不会误开启对话。

## Edge Cases

- 连续快速点击继续按钮不会跳出空节点或报错。
- 选择指向不存在节点时，对话控制器安全结束并打印错误日志。
- 同场景多个 `DialogueTrigger` 存在时，只能与当前范围内的目标交互。
- 对话被结束后再次接近 NPC，若允许重复对话则可重新触发。
